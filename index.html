ai-dev — Autonomous Coding Agent

From a GitHub issue to a tested, auto-merged pull request — autonomously, on local models.

ai-dev is a self-hosted agent. You open a GitHub issue, label it ai-dev, and it plans the work, writes the code, opens a pull request, runs CI, fixes its own failures, and auto-merges when tests pass — using local LLMs served by LM Studio, no cloud APIs.

How it works

1. Issue Trigger

A GitHub issue is opened and labeled ai-dev (only allowlisted users can trigger it).

2. Planning

The orchestrator parses the issue into a structured spec, then a step-by-step plan.

3. Implementation

It creates a branch feature/issue-<n>, implements the changes, commits, and pushes.

4. PR Creation

It opens a pull request; GitHub Actions runs the CI workflow.

5. Fixing

If CI fails: it fetches the CI logs, finds the root cause, applies a fix, and retries (up to 5 attempts).

6. Merge

When CI is green, it squash-merges the PR and can fire an optional deploy webhook.

This page itself was produced by that pipeline — issue → PR → green CI → merge.

Architecture & tech stack

Orchestrator: Node.js + TypeScript with an Express HTTP server.

GitHub: a GitHub App with webhooks (issues + workflow runs) via Octokit.

Ingress: webhooks arrive over a Cloudflare tunnel.

State: SQLite — each issue is a job tracked through a state machine (QUEUED → PARSING → PLANNING → IMPLEMENTING → PR_OPEN → CI_RUNNING → FIXING → MERGED / DEPLOYED / FAILED).

Git: clones, branches, commits, and pushes via the git CLI using short-lived tokens.

Observability: structured logs; every model call (model, prompt, response, latency) is recorded.

Packaging: runs as a Docker container next to the local model server.

Safety & guardrails

Max 5 fix attempts per issue; one active branch per issue.

Never auto-merges unless every CI check is green.

Repository allowlist (scoped to the org) and a trigger-user allowlist.

Signature-verified webhooks; file patches are path-traversal guarded.
